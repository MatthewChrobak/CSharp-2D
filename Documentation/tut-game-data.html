How to store data for your game

When making a game, it is incredibly important to store information about objects inside your game. For example, you might want to store detail about the state of your player (like health for example) so that the game knows when the player is dead or alive. Simply being able to store this information is not enough. We might need other functioning components of your game to access this data. For example, in the situation previously mentioned, the graphics system needs to know how much health the player has so that it may display it in a 'health bar'.

This tutorial will show you how to store data, and make it easy to access in other components of your game.

The models folder can be found in all templates in the Data folder. In this folder, we will create classes and structure them to store game data.

Create a new class inside the models folder, and name it whatever you like. For the sake of this tutorial, we will create a class called Player.

We will create fields in the player class to represent player attributes, such as name, age, and health. We will also make a constructor to initialize these fields with default values. 

				

public class Player
{
	public string Name;
	public int Age;
	public double Health;

	public Player() {
		this.Name = "Player1";
		this.Age = 0;
		this.Health = 100.0;
	}
}

Now that we have the structure of the player, let's make an instance of it in the Data Manager. 

				

// Remember to declare usage of the models namespace.
using Game.Data.Models;

namespace Game.Data
{
    public static class DataManager
    {
        // The object containing player-related data.
        public static Player Player;


        public static void Load() {
            // Inlcude all data-loading logic here.
        }

        public static void Save() {
            // Include all data-saving logic here.
        }
    }
}

Now to access the player, all you need to do is declare usage of the Data namespace, and reference the Player object through the DataManager class. 

				

// Remember to declare usage of the data namespace.
using Game.Data;

namespace Game
{
    public static class Program
    {
        public void Foo() {
            // Reference the player through the DataManager class.
            var player = DataManager.Player;
        }
    }
}
